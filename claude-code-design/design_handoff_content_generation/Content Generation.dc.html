prototype note — this is a local-development authoring tool. Edits here persist in your browser; in the real app each button hits a Next.js route that runs the matching script (parse:resume · generate:* with GEMINI_MODEL) and writes the files on your machine — then you commit them with git as usual. Editing only works on localhost: the unlock route checks the password against process.env.CONTENT_GENERATION_PASSWORD and refuses in production, so the deployed site stays read-only. No database — the password lives only on the server, never in the browser bundle.
